## Task C - Add SECURITY.md

<read_first>
- `docs/security/github-cache-purge-request.md`
- `.github/workflows/secret-guard.yml`
- `apps/kitchen/src/lib/content-scanner.ts`
- `README.md`
</read_first>

<action>
Create `SECURITY.md` covering:
1. Supported versions/current development status.
2. Responsible disclosure instructions without exposing private contact details beyond public repo guidance.
3. What counts as a security issue for Kitchen: A2A auth bypass, secret exposure, prompt/task injection, MCP/tool permission issues, registry/auth issues, memory data leakage.
4. What not to include in reports: real keys, private user data, exploit payloads against third-party systems.
5. Current security controls: per-agent API keys, operator key, secret guard workflow, content scanner, and documented env profile model.
</action>

<verify>
`test -f SECURITY.md && rg "responsible disclosure|API key|secret|A2A|memory" SECURITY.md`
</verify>

<acceptance_criteria>
- `SECURITY.md` exists at repo root.
- It does not include private email addresses, internal domains, or private infrastructure names.
- It clearly tells reporters not to include live secrets or sensitive personal data.
</acceptance_criteria>
