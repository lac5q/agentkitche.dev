# Phase 41 Plan 02: Public CI Release Gate

<objective>
Turn the current internal CI into a public release gate that explicitly runs typecheck, lint, tests, build, Python service tests, and a Docker compose smoke check on PRs.
</objective>

<threat_model>
## Threat Model

- Threat: CI logs expose real environment values or local paths. Severity: high. Mitigation: use placeholder env values and avoid printing `.env` contents.
- Threat: Docker compose smoke accidentally requires Luis-specific infrastructure. Severity: high. Mitigation: run with placeholder cloud-service values only when the service can boot without contacting paid/private services, or keep the smoke to config/build/health targets that can run safely in CI.
- Threat: CI becomes too slow or flaky for outside contributors. Severity: medium. Mitigation: separate fast lint/typecheck/tests from Docker smoke, use timeouts, and document required env.
</threat_model>

## Task C - Add Docker compose smoke path

<read_first>
- `docker-compose.yml`
- `docker/Dockerfile.kitchen`
- `docker/Dockerfile.memory`
- `docker/Dockerfile.orchestration`
- `docker/Dockerfile.voice`
- `docker/Dockerfile.knowledge`
- `.env.example`
</read_first>

<action>
Add `scripts/docker-compose-smoke.sh` that performs the least-flaky public smoke check:
1. Validates compose config with safe placeholder env values.
2. Builds the compose services that do not require live private API calls during build.
3. Starts services only if required env placeholders are enough for health checks, with a bounded timeout.
4. Always tears down compose resources on exit.

Wire the script into CI as the Docker compose smoke gate. If full service startup needs real Qdrant/Gemini credentials, make the script fail clearly in full mode and provide a CI-safe config/build-only mode.
</action>

<verify>
`bash -n scripts/docker-compose-smoke.sh && ./scripts/docker-compose-smoke.sh --config-only`
</verify>

<acceptance_criteria>
- Script exists and is executable.
- Config-only smoke works without private credentials.
- CI invokes the script.
- Qdrant remains cloud-only; no local Qdrant container is added.
</acceptance_criteria>
