# Phase 69 Plan 01: Context Source Contracts + Runtime Resilience

## Product Goal

Make Memoroos installs self-aware about every source of context they depend on.
Operators can see whether Gmail, Spark, qmd, mem0, and other enabled source
lanes are fresh, searchable, and safe to use. Setup installs the required
runtime monitors, recurring degradation evals prove the failure modes stay
covered, and source-backed answers fail closed when required source material is
stale or unavailable.

## Requirements Covered

- CTX-01: declarative context source contracts
- CTX-02: source runner validation
- CTX-03: context health API and UI
- CTX-04: stale-source safe-answer gate
- CTX-05: runtime service installer
- CTX-06: env and secret propagation
- CTX-07: degradation eval/UAT suite
- CTX-08: source contract docs and templates

## Implementation Plan

1. Add context source contract schema and config.
   - Create a product-owned source config file, for example
     `context-sources.config.json`.
   - Model source id, type, enabled flag, required tools, env vars, source path,
     ingest command, index command, freshness threshold, qmd collection, and
     safe-answer policy.
   - Ship default examples for `qmd`, `gmail`, `spark`, and `local-folder`
     sources without Luis-specific paths.

2. Add source runner validation.
   - Validate required binaries (`qmd`, `gws`, `sqlite3`, `ollama` where
     applicable).
   - Validate source paths and app-specific SQLite files before ingestion runs.
   - Validate Python/node runner availability without hardcoded virtualenv
     paths.
   - Produce actionable errors instead of non-fatal silent logs.

3. Add context health API.
   - Implement `/api/context/health`.
   - Return per-source status: `ok`, `stale`, `missing`, `degraded`, `disabled`.
   - Include last run, age, document count, qmd collection status, last indexed
     item marker, and last error.
   - Keep memory health (`mem0` queue/vector/disk/SQLite) linked but distinct
     from static source health.

4. Add operator UI.
   - Add a Context Sources panel under Library or Memory Intelligence.
   - Show each source lane with status, freshness age, searchability, and the
     exact command or setup action needed to repair it.
   - Surface warnings for enabled sources that are not indexed or stale beyond
     threshold.

5. Add stale-source safe-answer gates.
   - Introduce a small server-side helper that checks required source contracts
     before source-backed tasks.
   - For meeting minutes and email-context tasks, return a structured
     `SOURCE_STALE` or `SOURCE_MISSING` state when the required source is not
     fresh.
   - Document the agent behavior: do not reconstruct source-backed artifacts
     from adjacent docs unless the user explicitly requests a reconstruction.

6. Productize runtime service installation.
   - Extend the memory resilience installer pattern into a general runtime
     service installer.
   - Generate launchd plists from the install path, not hardcoded paths.
   - Cover mem0 server, qmd MCP, memory resilience, batch embedding/indexing,
     and optional source ingestion jobs where supported.
   - Add `status`, `install`, `uninstall`, and `check` commands.

7. Standardize env and secret propagation.
   - Ensure generated services read env files instead of embedding secrets in
     plist bodies.
   - Add `.env.example` fields for source contracts and alert sinks.
   - Add setup validation that reports missing secret/env requirements per
     enabled source.

8. Add degradation eval/UAT coverage.
   - Extend `npm run eval:memory-degradation` or add
     `npm run eval:context-sources`.
   - Cover scenarios:
     - mem0 queue backlog
     - embedding backend failure
     - qmd down
     - qmd collection stale
     - Gmail runner missing dependency
     - Spark source present but not indexed
     - source file exists but classified into wrong project
     - source-backed task requested while source stale
   - Assert API response, UI rendering, alert/log evidence, and safe-answer
     behavior.

9. Update documentation and templates.
   - Document how to enable/disable each source type.
   - Provide a "how to know my context is indexed" troubleshooting page.
   - Document privacy expectations: raw source docs stay in qmd/static
     knowledge by default; mem0 stores distilled durable facts.
   - Include a template for adding a new source contract.

## Completion Criteria

- A fresh macOS install can run setup and receive generated, path-correct
  runtime monitors without hand-editing launchd plists.
- At least Gmail, Spark, qmd, and mem0 have source/health contracts with
  visible status in the UI.
- Source-backed meeting or email tasks fail closed when the required source is
  stale, missing, or unindexed.
- The recurring eval suite covers the failure modes that caused the May 2026
  incidents.
- Docs explain how an operator can prove a source is ingested, indexed,
  searchable, and safe to use.
