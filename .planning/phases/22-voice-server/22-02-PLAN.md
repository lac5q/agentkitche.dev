**1. Extend `/api/recall/route.ts`** — Add `agent_id` query parameter support. Currently the route only accepts `q` and `limit`. Add:
```typescript
const agentId = url.searchParams.get('agent_id');
```
When `agent_id` is provided, filter the results by `agent_id` in the SQL query. When `agent_id` is present but `q` is empty, return all messages for that agent_id (not just FTS results) ordered by timestamp DESC, limited to `limitParam`. This is needed because `/api/recall?agent_id=voice` should return the voice transcript without requiring a keyword search. Read the existing `recallByKeyword` function in `src/lib/db-ingest.ts` to understand the query pattern, then add a new branch or helper that handles the agent_id-only case with a simple `SELECT * FROM messages WHERE agent_id = ? ORDER BY timestamp DESC LIMIT ?`.
