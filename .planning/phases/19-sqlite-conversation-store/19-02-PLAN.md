Create `src/lib/db-ingest.ts` with the following exports:

1. `deriveAgentId(projectDirName: string): string` — per Research Pattern 3:
   - If contains 'paperclip', return 'paperclip'
   - Otherwise decode by replacing leading `-` with `/`, then all `-` with `/`, split on `/`, return last non-empty component
   - Fallback: 'claude-code'

2. `extractContent(entry: JsonlEntry): string | null` — per Research Pattern 5:
   - For `type: 'user'`: if `message.content` is string, return `.slice(0, 8000)`. If array, filter for `type: 'text'` blocks, join `.text` with newline, slice to 8000.
   - For `type: 'assistant'`: if `message.content` is array, filter for `type: 'text'` blocks only (skip `thinking`, `tool_use`, `tool_result`), join `.text` with newline, slice to 8000.
   - For all other types (`system`, `file-history-snapshot`, etc.): return null.

3. `ingestFile(db: Database.Database, filePath: string, projectDirName: string): number` — parse JSONL file line by line:
   - Read file with `fs.readFileSync` (synchronous — matches better-sqlite3 pattern)
   - For each line: `JSON.parse`, call `extractContent`, skip nulls
   - Insert via prepared statement: `INSERT OR IGNORE INTO messages (session_id, project, agent_id, role, content, timestamp, cwd, git_branch, request_id) VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?)`
   - session_id = filename without extension
   - project = deriveAgentId(projectDirName) (reuse for agent_id too)
   - Wrap all inserts in a transaction for performance
   - Return count of inserted rows

4. `ingestAllSessions(db: Database.Database): { filesProcessed: number; rowsInserted: number; filesSkipped: number }`:
   - Use `fs.readdirSync` on `CLAUDE_MEMORY_PATH` to list project directories
   - For each project dir, list `.jsonl` files
   - For each file: check `ingest_meta` — if mtime_ms AND file_size match, skip
   - Otherwise call `ingestFile`, then upsert `ingest_meta` with new mtime_ms, file_size, row_count, ingested_at
   - After all files: update `meta` table `last_ingest_ts` to current ISO timestamp
   - Use `fs/promises` only for directory listing, NOT for file reading (keep better-sqlite3 sync pattern with `fs.readFileSync`)
   - Actually, use `fs.readdirSync` and `fs.statSync` and `fs.readFileSync` throughout — the entire function is synchronous since better-sqlite3 is synchronous. Return the result directly, not a Promise.

5. `recallByKeyword(db: Database.Database, query: string, limit: number = 20): RecallResult[]`:
   - Cap limit at 100 (DoS prevention per Research Security Domain)
   - First attempt: wrap query in double quotes for phrase match: `'"' + query + '"'`
   - If zero results: retry with plain query (no quotes)
   - SQL: `SELECT m.session_id, m.project, m.agent_id, m.role, snippet(messages_fts, 0, '<mark>', '</mark>', '...', 32) AS snippet, m.timestamp, rank FROM messages_fts JOIN messages m ON m.id = messages_fts.rowid WHERE messages_fts MATCH ? ORDER BY rank LIMIT ?`
   - Catch SQLite FTS5 syntax errors and return empty array (malformed queries)
   - Export `RecallResult` type

Create test file using a temp directory with mock JSONL files. The test should create actual JSONL files with realistic structure (user text entries, assistant text+thinking entries, tool_use entries) and verify the ingestion and recall pipeline end-to-end.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> /api/recall | User-supplied `q` parameter enters FTS5 MATCH clause |
| Filesystem -> DB | JSONL files from ~/.claude/projects/ read and parsed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-03 | Tampering | FTS5 query injection via `q` param | mitigate | Wrap query in double quotes for phrase match; catch SQLite syntax errors and return empty results; never interpolate raw query into SQL string |
| T-19-04 | Denial of Service | Unbounded limit param | mitigate | Cap `limit` at 100 server-side regardless of query param value |
| T-19-05 | Tampering | Path traversal in JSONL ingest | mitigate | Only read files under resolved CLAUDE_MEMORY_PATH; use readdirSync (no user-supplied paths) |
| T-19-06 | Information Disclosure | Large content in responses | accept | Content truncated to 8000 chars at ingest; snippets further truncated to 32 tokens by FTS5 snippet() |
</threat_model>
