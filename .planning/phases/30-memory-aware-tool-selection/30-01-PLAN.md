# Phase 30 Plan 01: Outcome-Aware Tool Selection

## Goal

Use the existing Tool Attention outcome log to bias discovery toward tools that have helped recently, while keeping private task text out of returned stats.

## Scope

- Summarize recent outcomes by capability id.
- Count uses, successes, failures, last outcome, last used timestamp, and a compact score.
- Attach outcome summaries to matching capabilities.
- Use outcome score as a tie-breaker/boost in `tool_discover`.
- Expose aggregate outcome stats through `tool_stats`.

## Non-Goals

- Do not call mem0 yet.
- Do not return raw prior task text in aggregate stats.
- Do not add an LLM scoring dependency.

## Verification

- Extend Knowledge MCP pytest coverage.
- Confirm aggregate stats omit raw task text.
- Confirm direct top-level MCP wrappers expose the same stats.
