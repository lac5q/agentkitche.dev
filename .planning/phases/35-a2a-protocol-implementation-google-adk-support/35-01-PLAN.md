# Phase 35 Plan 01: A2A Foundation, Config, And Memroos Agent Card

<objective>
Create the standards-first A2A foundation: config-derived operating-profile seams, spec-shaped types/errors, and Memroos's own public A2A agent card at the current canonical well-known path plus the roadmap compatibility alias.
</objective>

<threat_model>
## Threat Model

- Threat: Memroos publishes an agent card with localhost-only URLs, wrong security metadata, or stale protocol fields. Severity: high. Mitigation: derive URLs/security from `apps/memroos/src/lib/a2a/config.ts`, add tests for configured `MEMROOS_PUBLIC_BASE_URL`, and declare bearer/API-key security in the card.
- Threat: the public agent card leaks secrets, local filesystem paths, API keys, or private environment values. Severity: high. Mitigation: card builder allowlists public fields only and tests assert no `apiKey`, `token`, `secret`, `AGENT_CONFIGS_PATH`, or `KNOWLEDGE_BASE_PATH` strings appear in JSON.
- Threat: Phase 35 drifts from the current A2A spec and bakes stale roadmap names into source. Severity: medium. Mitigation: centralize operation/path constants in `types.ts` and route tests for `/.well-known/agent-card.json` as canonical plus `/.well-known/agent.json` as compatibility-only.
</threat_model>
