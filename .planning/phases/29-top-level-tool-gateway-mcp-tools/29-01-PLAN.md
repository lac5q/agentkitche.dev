# Phase 29 Plan: Top-Level Tool Gateway MCP Tools

## Goal

Make the v1.6 Tool Attention runtime directly callable by MCP clients without forcing agents through the generic `knowledge_workspace_call` meta-tool.

## Scope

- Add top-level Knowledge MCP tools:
  - `tool_catalog`
  - `tool_discover`
  - `tool_load`
  - `tool_record_outcome`
  - `tool_stats`
- Preserve the existing `tool-attention` workspace action path.
- Add tests that call the top-level functions directly.

## Implementation

1. Add thin wrappers in `services/knowledge-mcp/knowledge_system/mcp_server.py`.
2. Delegate all behavior to `services/knowledge-mcp/knowledge_system/tool_attention.py`.
3. Add pytest coverage in `services/knowledge-mcp/tests/test_knowledge_system.py`.

## Verification

- Run Knowledge MCP test suite.
- Confirm direct wrappers return public-safe catalog/load/discover/record/stats responses.
- Confirm outcome recording still redacts local root paths.
