# Phase 64 Plan 01: Immutable Audit + HIL Escalation

## Product Goal

Every agent decision, SEAL proposal lifecycle action, and eval run completion is appended to a single unified, immutable `audit_entries` table тАФ one query surface for finance clients and auditors. HIL escalations surface in a team-visible queue with configurable per-type SLA countdown and overdue flagging. CSV and JSON export work end-to-end. The existing `seal_audit_log` and `audit_log` tables are migrated in, and immutability is enforced at both the SQLite trigger layer and the service code layer.

## Requirements

- **AUDIT-01**: Immutable audit log тАФ every agent decision (match/flag/escalate), SEAL proposal action, and eval run is appended with actor, timestamp, reason; no deletes
- **AUDIT-02**: Audit log is queryable by agent, time range, decision type, actor
- **AUDIT-03**: Audit export тАФ CSV/JSON export for compliance review
- **AUDIT-04**: HIL escalation queue тАФ unresolved escalations surface in the UI with SLA countdown

### Task 2 тАФ Schema: `audit_entries` table with immutability triggers and indexes

**File:** `apps/memroos/src/lib/db-schema.ts`

Add the following to `initSchema` (additive only, after existing Phase 62 blocks):

```sql
CREATE TABLE IF NOT EXISTS audit_entries (
  id            TEXT PRIMARY KEY,
  tenant_id     TEXT NOT NULL DEFAULT 'default-tenant'
                REFERENCES tenants(id),
  actor_id      TEXT NOT NULL,
  actor_role    TEXT NOT NULL,
  event_type    TEXT NOT NULL,
  entity_type   TEXT NOT NULL,
  entity_id     TEXT NOT NULL,
  reason        TEXT,
  metadata_json TEXT NOT NULL DEFAULT '{}',
  created_at    TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ','now'))
);
CREATE INDEX IF NOT EXISTS audit_entries_created
  ON audit_entries(created_at DESC);
CREATE INDEX IF NOT EXISTS audit_entries_entity
  ON audit_entries(entity_type, entity_id, created_at DESC);
CREATE INDEX IF NOT EXISTS audit_entries_event_type
  ON audit_entries(event_type, created_at DESC);
CREATE INDEX IF NOT EXISTS audit_entries_actor
  ON audit_entries(actor_id, created_at DESC);
CREATE INDEX IF NOT EXISTS audit_entries_tenant
  ON audit_entries(tenant_id, created_at DESC);
CREATE INDEX IF NOT EXISTS audit_entries_tenant_event
  ON audit_entries(tenant_id, event_type, created_at DESC);

CREATE TRIGGER IF NOT EXISTS audit_entries_no_update
  BEFORE UPDATE ON audit_entries
BEGIN
  SELECT RAISE(ABORT, 'audit_entries is append-only: UPDATE is not permitted');
END;

CREATE TRIGGER IF NOT EXISTS audit_entries_no_delete
  BEFORE DELETE ON audit_entries
BEGIN
  SELECT RAISE(ABORT, 'audit_entries is append-only: DELETE is not permitted');
END;
```

### Task 5 тАФ Unified audit write service

**File:** `apps/memroos/src/lib/audit/write.ts`

Implement three exported functions:

**`writeAuditEntry(entry: NewAuditEntry, db?: Database): void`**
- INSERT only тАФ no UPDATE or DELETE statements in this module
- Generates UUID v4 `id` if not provided
- Injects `created_at` if not provided
- Throws with a descriptive error if required fields are missing (type-level enforcement via `NewAuditEntry` type)

**`openEscalation(params: NewHilEscalation, db?: Database): string`** (returns new escalation ID)
- Resolves `sla_seconds` from `memroos.eval.yaml` config by `escalation_type` key
- Computes `sla_deadline = new Date(Date.now() + sla_seconds * 1000).toISOString()`
- Wraps in `db.transaction()`: INSERT into `hil_escalations` + `writeAuditEntry({ event_type: 'hil.created', entity_type: 'hil_escalation', entity_id: 'hil_escalation:' + id, ... })`
- Returns new escalation `id`

**`resolveEscalation(id: string, resolution: { actorId: string; actorRole: string; note?: string }, db?: Database): void`**
- `db.transaction()`: UPDATE `hil_escalations SET status = 'resolved', resolved_by = ?, resolution_note = ?, resolved_at = ?` + `writeAuditEntry({ event_type: 'hil.resolved', ... })`
- Note: `hil_escalations` IS mutable (it tracks open-item state); only `audit_entries` is immutable
- Throws if escalation is not in `open` or `sla_breached` status (cannot resolve an already-resolved item)

### Task 6 тАФ Audit query and SLA service

**File:** `apps/memroos/src/lib/audit/query.ts`

- `queryAuditEntries(filter: AuditQueryFilter, db?: Database): { entries: AuditEntry[]; nextCursor?: string }` тАФ paginated query; builds parameterized WHERE clause from filter; returns up to `filter.limit` (max 200) rows ordered by `created_at DESC`; cursor is the `created_at` of the last row

**File:** `apps/memroos/src/lib/audit/sla.ts`

Implement:
- `checkSlaBreaches(db?: Database): number` тАФ queries `hil_escalations WHERE status = 'open' AND sla_deadline < datetime('now')`; for each: UPDATE status to `sla_breached` + write `hil.sla_breached` audit entry; returns count of newly breached items; this function is called lazily on each `GET /api/escalations` response

## Success Criteria

Each item must be verifiably TRUE before Phase 64 is marked complete.

1. `audit_entries` table is append-only тАФ `UPDATE` and `DELETE` both raise the SQLite trigger error; no UPDATE or DELETE SQL statements exist in any source file under `apps/memroos/src/lib/audit/`

2. Every agent match/flag/escalate decision, SEAL apply/rollback, and eval run completion writes an `audit_entries` row with `actor_id`, `event_type`, `entity_type`, `entity_id`, and `created_at` populated

3. All four filter patterns (by agent, time range, event type, actor) return correct results; perf test confirms p95 < 200ms on 1M rows for each pattern

4. `GET /api/audit/export?format=csv` and `GET /api/audit/export?format=ndjson` return complete, valid output for a non-empty filter query

5. Open HIL escalations appear in the `/escalations` UI queue with a visible SLA countdown; items past their `sla_deadline` are flagged red with `sla_breached` status

6. `POST /api/escalations/:id/resolve` with operator or admin JWT succeeds and writes an `hil.resolved` audit entry; the same call with reviewer JWT returns 403

7. Phase 63's `users.id` is used as `actor_id` for all human-initiated audit entries; automated events use `actor_id = 'system'`

8. Backfill migration runs once (guarded by `meta` flag); `seal_audit_log` and `audit_log` legacy rows are present in `audit_entries` after migration
