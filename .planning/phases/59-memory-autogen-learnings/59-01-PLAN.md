## Product Goal

Closed-loop memory self-improvement: SEAL-style typed proposals that mutate the
memory subsystem, plus a fixed-harness "policy lab" that ranks memory-policy
variants by composite W (Karpathy autoresearch style).

## As-built Surface

- `lib/memory-policy-lab.ts` — injectable-evaluator policy ranker over a golden
  set; ranks variants by composite W via `scoreTraceWithEvalEngine`.
- `lib/memory-recall-evals.ts` — recall fixtures across vector/graph/episodic/qmd
  tiers and timings; canary/gold/full modes.
- `app/api/memory/evals/{run,latest}` — run + latest-result endpoints.
- `app/api/memory/policy-lab` — policy-lab run endpoint.
- `app/api/memory/proposals` — list/filter the 5 memory proposal types.
- `app/memory-autogen/` + `components/memory/` — UI (proposals panel, policy lab).
- `scripts/run-memory-evals.mjs` — `npm run eval:memory` harness.
