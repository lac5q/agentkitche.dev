**Context:** Gwen is the OpenClaw agent instance (🌸). OpenClaw doesn't have self-improving natively. Hermes (OpenCode) has this natively via her 6-step heartbeat cycle. Gwen needs it via a skill.

**Install:** `clawdhub install self-improving-agent` in Gwen's OpenClaw workspace

**Configuration constraints (must be enforced):**
1. **Memory writes → mem0 only.** All learned memories must go through `POST http://localhost:3201/memory/add` with `agent_id="gwen"`. Do NOT enable Cognee-OpenClaw — it creates a parallel memory layer that conflicts with Qdrant/mem0 architecture.
2. **Skills → staging only.** Any skills Gwen creates must be written to `~/github/knowledge/skills/.hermes-staging/<skill-name>/` so Hermes's `skill-sync.py` picks them up. Not to `~/skills/` (ClawHub silo) or any other location.
3. **No workspace prompt injection** that bypasses the SessionStart mem0 preload hook.
4. **Reflection cron timing:** Schedule at 3am (not 4am — avoid collision with Hermes's sync cron).
5. **Skip Cognee-OpenClaw** — explicitly do not install the "advanced memory layering" addon.

**What Gwen gains:** Captures learnings from sessions, creates reusable skills from workflows, builds persistent memory via mem0, reflective loop after tasks — same capability Hermes has natively.
