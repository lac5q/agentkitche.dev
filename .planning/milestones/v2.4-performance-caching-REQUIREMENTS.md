# Requirements: Agent Kitchen v2.4 — Performance + Caching

*Created: 2026-05-11*

## Goal

Improve response latency and cold-start behavior after security and model quality instrumentation exist.

## Requirements

### Phase 53 — Response Caching Layer

**CACHE-01** — API route cache: Cache safe route responses with configurable TTLs.

**CACHE-02** — Knowledge MCP cache: Cache search/discover results by query hash and invalidate on index updates.

**CACHE-03** — Memory cache: Cache memory recall by query, agent, collection, and tier; invalidate on writes.

**CACHE-04** — Graph cache: Cache Neo4j queries with tag-based invalidation.

**CACHE-05** — A2A task cache: Cache task status per task ID and invalidate on status transition.

**CACHE-06** — Cache stats endpoint: Expose hit/miss ratio, entry counts, and memory usage.

**CACHE-07** — Cache UI: Show cache health, stale entries, and manual purge controls.

### Phase 54 — Query Performance + Cold Start Elimination

**PERF-01** — Cache pre-warm: Pre-warm registry, skills, tool attention, memory health, and model-routing caches.

**PERF-02** — Latency budgets: Define and enforce p95 budgets for cache-hit and cache-miss routes.

**PERF-03** — SQLite optimization: Verify query plans for recall and orchestration checkpoint lookups.

**PERF-04** — Neo4j optimization: Add indexes and bounded traversal patterns for common graph queries.

**PERF-05** — Regression detection: CI fails on material latency degradation.

## Non-Goals

- Adding Redis.
- Adding local Qdrant to Docker compose.
- Optimizing model quality or routing; that is v2.2.
