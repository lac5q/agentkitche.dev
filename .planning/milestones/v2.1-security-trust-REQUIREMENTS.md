# Requirements: Memroos v2.1 — Security + Trust Layer

*Created: 2026-05-11*

## Goal

Harden Memroos's public agent surfaces before expanding optimization or performance work. v2.1 protects A2A, REST shim, Dispatch UI, tool/delegation policy, and memory write paths.

`4d03fae` already shipped the first security foundation: Agent Shield/Iris planning notes, `iris-scanner.ts`, prompt-injection pre-flight rules, Dispatch/A2A wiring, and tests. Requirements below mark that foundation as shipped and make the tool permission guard the next implementation target.

## Requirements

### Phase 42 — Agent Shield + Iris Pre-flight Foundation

**SEC2-01** — Reproducible scan: Run Agent Shield against Memroos and Hermes and store summarized findings.

**SEC2-02** — Finding triage: Classify findings into true positive, false positive, accepted risk, and follow-up.

**SEC2-03** — Gap analysis: Document scanner categories not covered by `content-scanner.ts`.

**SEC2-04** — Integration decision: Decide dependency, fork/adapt, or custom rule-port path with license and maintenance notes.

**IRIS-01** — Pre-flight task scanner: A2A, REST shim, and Dispatch UI task ingress call a shared scanner.

**IRIS-04** — Redacted audit events: Iris decisions write structured redacted audit events.

**IRIS-05** — Compatibility: Existing safe dispatch and registration flows continue to pass.

### Phase 43 — Tool Permission Guard + Policy Enforcement

**POLICY-01** — Capability enforcement: Tool calls and A2A delegation check canonical registry capabilities.

**POLICY-02** — Network policy: Operating profiles enforce allowed remote-card and delegation destinations.

**POLICY-03** — Memory policy: Memory writes enforce allowed tier and agent identity.

**POLICY-04** — Redacted denials: Denied actions return structured errors without leaking secrets.

**POLICY-05** — Policy auditability: Policy decisions are persisted and queryable.

**Implementation note:** Local Phase 43 implementation enforces these with `apps/memroos/src/lib/security-policy.ts`, `/api/dispatch`, `sendA2aMessage`, and `/api/memory/add`. Agents with no declared capabilities remain allowed for backward compatibility; declared-but-insufficient capabilities are denied.

### Phase 44 — Security Operations UI + Reports

**SECUI-01** — Security event UI: Show Iris, policy, secret-scan, and Agent Shield events.

**SECUI-02** — Filters: Filter events by severity, agent, route, task, and action.

**SECUI-03** — Security health: Show critical/high open findings and scan freshness.

**SECUI-04** — Report generation: Generate redacted Markdown security reports for release review.

### Phase 45 — Progressive Security Capability Exposure

**SECCAP-01** — Progressive capability: Expose `capability:agent-shield` or equivalent Iris security capability through discovery.

**SECCAP-02** — Policy mode config: Agents can declare or inherit strict, standard, or permissive security mode.

**SECCAP-03** — Registry visibility: Agent security mode and capability status are visible without exposing secrets.

**SECCAP-04** — Graceful degradation: Capability checks degrade cleanly when Agent Shield or optional scanners are unavailable.

### Deferred Security Follow-Ups

**IRIS-02** — Policy modes for Iris block/flag/pass behavior are handled by Phase 45 security mode configuration.

**IRIS-03** — Post-execution output scanning remains a follow-up once the tool guard/audit surfaces settle.

## Non-Goals

- Multi-tenant RBAC.
- Full SOC/compliance workflow.
- Replacing existing content scanner in one step.
- Performance/caching work; that is v2.4.
