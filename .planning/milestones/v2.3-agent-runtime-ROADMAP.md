## Phase 50: Agent-Side Middleware

**Goal:** Add a middleware layer to Hermes agent tool calls that enables pre-call hooks (input validation, redaction) and post-call hooks (outcome logging, skill health alerts), making the agent system self-healing.

**Requirements:** MWMID-01, MWMID-02, MWMID-03, MWMID-04, MWMID-05, MWMID-06

**Success Criteria (what must be TRUE):**
1. `~/.hermes/middleware/pre/` and `~/.hermes/middleware/post/` directories exist after setup
2. Middleware chain executes in alphabetical order for every tool call (native, MCP, shell, browser)
3. Input validation rejects calls with missing required fields before tool execution
4. Secret redaction works for patterns defined in `secret-patterns.json`; tool receives original value
5. Tool outcome logging produces one JSONL line per call with: tool name, input hash, success/failure, error type, duration_ms
6. Skill health alerts fire after 3 consecutive failures per skill (deduplicated); success resets counter
7. `middleware.enabled: false` skips all hooks without error
8. `middleware.skip` list correctly excludes specified middleware
9. Total middleware overhead < 50ms for a typical tool call
10. No breaking changes to existing tool calls

**Plans:** TBD
**UI hint:** no (agent-side only)

**Local implementation notes:**
- Added `apps/memroos/src/lib/agent-runtime/middleware.ts`.
- Runtime bootstraps `.hermes/middleware/pre`, `.hermes/middleware/post`, and `.hermes/logs`.
- Default hooks run in alphabetical order: validation, redaction, outcome logging, skill health.
- Supports `middleware.enabled` and `middleware.skip`.

## Phase 51: Agent Memory Client v2

**Goal:** Replace file-based memory with vector-backed structured memory supporting semantic search, relevance scoring, auto-compression, and TTL expiration — while maintaining backward compatibility with the existing `memory` tool API.

**Requirements:** MEMCLI-01, MEMCLI-02, MEMCLI-03, MEMCLI-04, MEMCLI-05

**Success Criteria (what must be TRUE):**
1. Semantic search returns relevant memories for queries with no exact keyword overlap
2. Context injection at session start selects top-N memories based on session topic
3. Total injected memory text ≤ 1,500 characters
4. Saving a duplicate memory merges it (merge_count increments)
5. Memories with TTL=1 day purged within 24h by cron; expired memories archived before deletion (30-day grace)
6. Existing `memory(action='add', ...)` calls work without modification
7. All existing memories migrated to new format on first run
8. No memory data lost during migration
9. Session start with memory injection completes in < 2 seconds

**Plans:** TBD
**UI hint:** no (agent-side only)

**Local implementation notes:**
- Added `apps/memroos/src/lib/agent-runtime/memory-client.ts`.
- Supports relevance search, context injection capped to 1,500 characters, duplicate merge counting, TTL archive/purge, and backward-compatible `memoryTool` calls.
- Uses local file storage for the agent-runtime slice without introducing external services.

## Phase 52: Agent Observability Dashboard

**Goal:** Build a visual execution tracing dashboard for Hermes agent sessions — showing tool call timelines, token usage, decision points, error rates, and session health — built on v2.1 security/audit data, v2.2 LLM telemetry, and Phase 50 outcome logs.

**Requirements:** OBSUI-01, OBSUI-02, OBSUI-03, OBSUI-04, OBSUI-05

**Success Criteria (what must be TRUE):**
1. Dashboard is a static HTML page served locally on the Mac; works offline
2. Can open dashboard, see last 20 sessions, click into any session to see full timeline
3. Dashboard shows token totals per session and per tool type
4. Decision points appear on timeline with agent's stated reasoning
5. Error rate chart shows 1h/24h/7d windows; clicking a spike reveals failing sessions
6. Sessions with health < 50 are flagged with reason
7. Data sources: v2.1 security/audit events + v2.2 model telemetry + Phase 50 outcome logs + existing session logs
8. No external dependencies beyond what's already installed
9. Dashboard loads in < 3 seconds on the Mac
10. No data written to external services

**Plans:** TBD
**UI hint:** yes (static HTML dashboard)

**Local implementation notes:**
- Added `apps/memroos/src/lib/agent-runtime/observability.ts`.
- Added `/api/agent-runtime/observability` serving offline static HTML.
- Dashboard reads Hermes `tool-outcomes.jsonl`, shows last sessions, tool timeline, error rate, and health score.
