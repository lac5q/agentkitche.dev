# Requirements: Agent Kitchen v1.7 — Progressive Tool Gateway Runtime

*Created: 2026-05-01*

## Goal

Turn v1.6 Tool Attention from a visible catalog into a callable runtime gateway that agents can use directly, learn from, and operate safely from Kitchen.

## Requirements

### Phase 29 — Top-Level Tool Gateway MCP Tools

- [x] **TOOLGW-01** — Expose `tool_catalog`, `tool_discover`, `tool_load`, `tool_record_outcome`, and `tool_stats` as top-level Knowledge MCP tools.
- [x] **TOOLGW-02** — Preserve the existing `knowledge_workspace_call("tool-attention", ...)` compatibility route.
- [x] **TOOLGW-03** — Add tests proving direct MCP tool wrappers call the same public-safe Tool Attention runtime.

### Phase 30 — Memory-Aware Tool Selection

- [x] **MEMGW-01** — Summarize recorded outcomes by capability id, result label, and recency.
- [x] **MEMGW-02** — Add outcome-aware ranking to `tool_discover` without exposing private task text.
- [ ] **MEMGW-03** — Add "similar task used these tools" recommendations backed by memory/searchable outcome metadata.

### Phase 31 — Kitchen Tool Gateway Operations UI

- [ ] **UIGW-01** — Add a Tool Gateway operations view with health, degraded sources, recent outcomes, and recommendation rationale.
- [ ] **UIGW-02** — Add filters for capability type, status, source, and top-level/runtime availability.
- [ ] **UIGW-03** — Expand the Flow Tool Gateway node detail panel with outcome trends and skipped/loaded summaries.

### Phase 32 — Gateway Hardening and Lint Cleanup

- [ ] **OPSGW-01** — Add CI coverage for the top-level Knowledge MCP gateway tools.
- [ ] **OPSGW-02** — Fix full-repo lint debt enough to enforce `npm run lint`.
- [ ] **OPSGW-03** — Resolve or isolate the production Turbopack NFT warning.

## Out of Scope

- Adding `mcp-agent` as a runtime dependency.
- Moving private Knowledge Hub content into this monorepo.
- Multi-user authorization.
